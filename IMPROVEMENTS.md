# NsCDE-zh 项目改进建议报告

本报告针对 NsCDE-zh (Not so Common Desktop Environment - Chinese) 的当前实现架构、用户体验及开发工作流进行了深度分析，并提出以下改进方案。

---

## 1. UI 与本地化架构优化 (UI & Localization)

### 1.1 逐步淘汰 FvwmScript 的硬编码布局
*   **现状**：目前大量 GUI 对话框（如 `NsCDE-ColorMgr`, `NsCDE-FontMgr`）基于 `FvwmScript` 编写。其组件坐标和窗口尺寸为硬编码，导致中文翻译稍长即会发生 UI 溢出或截断。
*   **建议**：
    *   **迁移至 PyQt5**：由于项目中已将 PyQt5 列为依赖，建议将复杂的管理工具从 `FvwmScript` 迁移到 PyQt 编写。Python 能提供更好的布局管理器（Layout Managers），自动根据字体大小调整窗口尺寸。
    *   **动态模板化**：对于仍需保留的 `FvwmScript`，可引入预处理器（如 `sed` 或自定义 Python 脚本），在编译阶段根据当前语言的平均字符宽度动态计算坐标。

### 1.2 本地化自动化校验
*   **建议**：在 `po/` 目录下建立自动化测试脚本，利用 `msgfmt --check` 之外，增加对关键 UI 字符串长度的静态分析预警，防止长文本破坏界面。

---

## 2. 字体与高分屏 (HiDPI) 适配

### 2.1 自动 DPI 探测与配置
*   **现状**：`README.md` 中提到用户可能需要手动执行 `sed` 命令来修改 `~/.NsCDE/Xdefaults.fontdefs` 中的 DPI。
*   **建议**：
    *   **启动脚本增强**：在 `nscde` 启动脚本中增加自动探测逻辑（通过 `xdpyinfo` 或 `xrandr` 计算实际 DPI），并自动根据探测值更新配置，减少用户手动干预。
    *   **多屏缩放支持**：针对现代工作站的多显示器环境，增加对不同屏幕设置不同缩放比例的支持（虽然受限于 X11，但可通过虚拟分辨率模拟）。

---

## 3. 工程化与代码质量

### 3.1 脚本现代化与静态检查
*   **现状**：项目包含大量 `ksh93` 和 `bash` 脚本。
*   **建议**：
    *   **ShellCheck 集成**：在 CI 工作流中引入 `shellcheck` 对 `nscde_tools/` 下的所有脚本进行扫描，修复潜在的变量引用和错误处理问题。
    *   **Python 类型检查**：为 `lib/python/` 目录下的核心逻辑添加 `mypy` 类型标注，提高主题引擎的维护性。

### 3.2 依赖管理简化
*   **建议**：
    *   **AppImage / Flatpak 支持**：NsCDE 依赖项繁多且对版本有要求。提供 Flatpak 包可以将所有依赖（包括特定的 `ksh93` 版本）打包，实现“一键启动”，降低新手安装门槛。

---

## 4. 核心功能增强

### 4.1 现代组合器集成 (Compositor Integration)
*   **建议**：虽然 NsCDE 追求复古感，但现代用户常需阴影、透明度等特性。可以考虑可选集成 `picom`，并提供预设的复古风格配置（如无阴影、仅提供基本透明度支持）。

### 4.2 增强的主题同步
*   **建议**：
    *   **Flatpak 应用主题覆盖**：目前主题引擎可能无法直接覆盖沙盒内的 Flatpak 应用。建议研究通过 `~/.var/app` 路径注入 GTK 主题的方案。

---

## 5. 改进计划优先级 (Roadmap)

| 优先级 | 任务 | 目标 |
| :--- | :--- | :--- |
| **P0 (极高)** | 自动 DPI 探测脚本 | 解决中文显示方框和模糊的头号问题 |
| **P1 (高)** | CI 中引入 ShellCheck/Flake8 | 确保脚本健壮性，减少安装崩溃 |
| **P1 (高)** | `po` 文件长度静态预警 | 优化本地化质量 |
| **P2 (中)** | 将一个核心管理工具迁移至 PyQt5 | 验证 UI 现代化方案的可行性 |
| **P3 (低)** | Flatpak 封装 | 扩展分发渠道 |
